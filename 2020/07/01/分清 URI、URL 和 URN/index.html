分清 URI、URL 和 URN

URI标准

RFC3986，即“Uniform Resource Identifier (URI)：Generic Syntax”，是一个 Internet Standard。

比如RFC 3986的URI组成部分：

  • 方案名 (https)
  • 域名 (tools.ietf.org)
  • 标识符 (/html/rfc3986)

标识符指的是实现区分唯一资源的方式(例如姓名、地址、上下文)，着重的是方式

标准化过程

先给出自己总结出的结论：URI是为了起到名称和定位器的作用，进一步分为定位器(URL)、名称(URN)，或者二者兼具。

URI是RFC1630提出的Informational RFC，未获得社区的任何认可。在实现标准化过程中，由于从外观上看更像是一个位置，而不是一个名称，从而成了通常所说的URL。而另一个URN也运营而生了。

  • URL涉及的是 URI 的子集，除识别资源外，它还通过描述其最初访问机制（比如它的网络“位置”）来提供定位资源的方法。

  • URN在历史上曾用于引用“urn”方案 [RFC2141] 下的 URI，这个 URI 需要是全球唯一的，并且在资源不存在或不再可用时依然保持不变，对于其他任何拥有名称的一些属性的 URI，都需要使用这样的 URI。

理解

URI是给出的抽象概念，而URL,URN是根据时代推移而来的具体实现方式。

因此未来的规范和相关的文档应当使用通用术语“URI”，而不是使用有更多限制的条目“URL”和“URN” 。
